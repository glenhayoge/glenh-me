# Introduction

Analog is a fullstack meta-framework for building applications and websites with [Angular](https://angular.dev/).

Similar to other meta-frameworks such as Next.JS, Nuxt, SvelteKit, Qwik City, and others, Analog provides a similar experience, building on top of Angular.

## Features[​](https://analogjs.org/docs#features "Direct link to Features")

- Supports Vite/Vitest/Playwright
- [File-based routing](https://analogjs.org/docs/features/routing/overview)
- [Support for using markdown as content routes](https://analogjs.org/docs/features/routing/content)
- [Support for API/server routes](https://analogjs.org/docs/features/api/overview)
- Hybrid [SSR](https://analogjs.org/docs/features/server/server-side-rendering)/[SSG support](https://analogjs.org/docs/features/server/static-site-generation)
- Supports Angular CLI/[Nx workspaces](https://analogjs.org/docs/integrations/nx)
- Supports [Angular components](https://analogjs.org/docs/packages/astro-angular/overview) with Astro

## Creating a New Application[​](https://analogjs.org/docs/getting-started#creating-a-new-application "Direct link to Creating a New Application")

To create a new Analog project, you can use the `create-analog` package with your package manager of choice:

```bash
npm create analog@latest
```

```bash
yarn create analog
```

```bash
pnpm create analog
```

### Serving the application[​](https://analogjs.org/docs/getting-started#serving-the-application "Direct link to Serving the application")

To start the development server for the application, run the `start` command.

```bash
npm run start
```

```bash
yarn start
```

```bash
pnpm start
```

Visit [http://localhost:5173](http://localhost:5173/) in your browser to view the running application.
